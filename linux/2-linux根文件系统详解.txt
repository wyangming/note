linux根文件系统详解

linux没有注册表

/boot：系统启动相关的文件，如内核、initrd，以及grub(bootloader)
/dev：设备文件
	块设备：随机访问，按对应的数据块进行访问
	字符设备：线性访问，按字符为单位
	设备号：主设备号(major)和次设备号(minor)
/etc：配置文件
/home：用户的家目录，每一个用户的家目录通常默认为/home/USERNAME
/root：root用户的家目录，生产环境不允许用root用户直接登录
/lib：库文件分为静态库(.a)与动态库(或者共享对象即.so shell object)两种。
	/lib/modules：内核模块文件
/media：挂载点目录
/mnt：额外的临时文件系统
/opt：可选目录，第三程序的安装目录
/proc：伪文件系统，本身为空的，系统启动起来不为空，是系统内核的映射文件。用于系统的调优工作
/sys：伪文件系统，跟硬件设备相关的属性文件。主要用于系统的调优
/tmp：临时文件，如果下面的文件一个月系统会自动清理里面的文件，每个用户都可以在里面创建文件，只可以删除自己创建的文件不可以删除其他用户的文件(除root)
/var：可变化的文件都放在这里。刚装完系统很小，随着系统的运行会越来越大
	cache：缓存
	lock：锁文件
	log：日志文件
	mail：邮件转存文件
	run：关于运行进程的文件
	tmp：临时文件目录
/bin：可执行文件，用户命令，系统自身运行起来基本运行所需的命令
/sbin：可执行文件，管理命令，系统自身运行起来基本运行所需的命令
/usr(share, read-only)：全局共享的只读文件
	bin：系统自身启动起来后为了提供一些基本功能所需的命令
	sbin：系统自身启动起来后为了提供一些基本功能所需的命令
	lib：如果/usr/bin与/usr/sbin所依赖的库没有在/lib下的话都会放到这个下面
	
	local：与系统无关的信息，以下文件是第三方所需的命令与库。可能与业务有关系的数据库

		sbin：
		lib：

文件级文件目录的命名规则：
	长度不可以超过255个字符
	不可用/做文件名
	严格区分大小写

目录管理
	ls
	cd
	pwd
	mkdir：创建空目录
		参数：
			-p：如果创建目录的上级目录没有创建的话也创建上级目录
			-v(verbose)：详细信息
		用法：
			mkdir -pv /root/m/n/q    执行结果为显示创建root下m目录n目录q目录
			mkdir -pv /root/m/n/{q,e}    同时创建/root/m/n/下的q与e两个目录
	rmdir(remove directory)
		只可以删除空目录
		参数：
			-p：可以删除所给的所有空目录，但是只可以删除一脉单传的空目录
			
文件创建与删除
	touch：可以用来创建空文件(主要用来是改时间戳的，如果文件不存在就创建文件)
		参数：
			-c：如果文件不存在则不创建文件，命令默认是创建文件
			-a：改变文件的访问时间，但文件的更改时间也会改变
			-m：改变文件的修改时间(modification)时间，同样也会修改文件的修改时间
			-t：把文件属性时间改为指定时间
		用法：
			touch -a a
			touch -m a
			touch -m -t 201212121212 a  把文件的修改时间改为2012年12月12日12点12分
	stat：显示文件与文件状态的信息
	用文本编辑器来创建文件
		ASCII：美国国家标准信息交换代码
			128不同字符：
				二进制：
					2^6=0,63
					2^7=0,127
					变化从000 0000 - 111 1111
		GBK：			
					2^16=65536(来表示字符)
		字符标准：GBK,GB2312,Unicode
		
		nano：是一个文本编辑器，可以用来创建文件
		file：也可以创建文件
	rm：删除文件，对于root用户来说rm相当于rm -i的别名，想用原生态的时候可以\rm来使用原生态命令
		参数：
			-i：删除的时候提示是否要删除
			-f：强行删除文件，不要提示
			-r：用来删除目录的时候可以递归删除目录下的所有文件
		用法：
			rm -rf /mnt/test 递归删除/mnt/test目录下的所有文件

复制与移动文件

	cp:复制文件
		cp sourfile targetfile1
			参数：
				-r：复制目录到另一个目录
				-i：如果复制的话目录文件存是不是需要提示覆盖
				-f：强行覆盖
				-p：复制的时候保留文件的属性
				-a：归档复制常用于备份
				也支持{}进行复制
			用法：
				cp /etc/{passwd,inittab,rc.d/rc.sysinit} /tmp/
	mv：移动文件(可以用于重命名文件)
		mv src dest
		mv -t dest src
	install：复制文件并设置属性，复制过去后有文件的执行权限
		参数：
			-d：DIRECOTRY...创建目录
			-t：DEST SRC...把多个文件复制到目录下去
		用法：
			install /ect/passwd /var/tmp/install.file1

cat：连接并显示，可以同时显示两个文件
	cat file1 file2 ...
	参数：
		-n：显示行号与文件内容无关
		-E：显示行尾的结束符(linux每一行的行结束符为$符，windows的行结束符为$+回车)
	使用：
		Shift+pageup或Shift+pagedown可以进行翻屏，对于一些大文件来说是查看不全，因为cat内容是在内存里的
tac：显示信息与cat显示信息的顺序相反
Ctrl+c键可以退出命令，如果命令不想执行也可用它再换一行另写命令

more:显示文件内容
	用法：
		空格键向后翻一屏
		b键向前翻一屏
		回车键一行一行向后翻
	如果翻到文件尾部就不可以向前翻了

less：显示文件内容
	用法：跟man命令显示内容的用法是一样的

head：查看文件的前n行，默认n是10
	参数：
		-n：显示前多少行
	用法：
		head -n 1 /etc/inittab
		head -2 /etc/inittab

tail：查看文件的后n行，默认n是10
	参数：
		-n：显示后多少行
		-f：显示完后没有退出，如果其他进程打开文件后在文件尾部追加的信息可以马上显示
	用法：
		tail /etc/inittab
		tail -n 1 /etc/inittab

文本处理
cut：文本剪切，不会影响文本在源文件中的位置
	参数：
		-d：指字分格符，默认是空格
		-f：显示第几个字段，后面不跟空格，如果要显示多个如下：
			-f 1,3 第1个与第3个字段
			-f 1-3 第1个到第3个字段
	用法：
		cut -d : -f1 /ect/passwd    显示/etc/passwd里的第一个字段
join：文本合并
sed：文本处理
awk：文本处理，报告生成，相当于一个编程语言
sort：对文本显示进行排序，并不会影响文本在源文件中的次序，是根据一个字符在ASCII里的排序进行的
	参数：
		-n：按数值大小进行排序
		-r：反向排序
		-t：字段分隔符
		-k：指定字段排序
		-u：排序后相同的行只显示一次
		-f：排序时忽略字符大小写
	用法：
		sort aa.txt
		sort -t: -k3 /ect/passwd   把/ect/passwd以第3个字段时行排序

uniq：只有相邻的重复行算是重复行
	参数：
		-d：只显示重复的行，重复的只显示一行
		-D：显示所有重复的行，如果两个重复则显示两行
		-c：显示某一行重复的次数

wc(word count)：统计一个文件中一共有多少行，多少个字符(空格也算字符)也就是单词数，多少个字节
	参数：
		-l：行数
		-w：单词数
		-c：字节
		-m：字符
		-L：最长的一行包含多少个字符
		
tr：转换或删除字符
	参数：
		-d：删除后面出现在字符
	用法：
		tr 'ab' 'AB'   把输入内容的信息里有ab的换为AB显示
		tr 'ab' 'AB' < /etc/passwd    把/etc/passwd文件里所有的字符a与b换成A与B
		tr 'a-a' 'A-B' < /etc/passwd    把/etc/passwd文件里所有的字符换成大写字符
		tr -d 'ab' < /etc/passwd    删除出现的/etc/passwd文件里出现的ab字符
文本三大利器：sed, awk, grep

！号加命令前几个单词可以执行上次与之匹配的命令
